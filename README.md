# Найбільший однорядний цифровий пазл

Скрипт на Node.js, який складає найдовшу можливу послідовність із набору
6-значних цифрових фрагментів, з'єднуючи їх за правилом: **останні 2 цифри
одного фрагмента = перші 2 цифри наступного**. Кожен фрагмент можна
використати не більше одного разу.

## Як це працює

1. **Граф.** Кожен фрагмент — це орієнтоване ребро в графі, де вузли — це
   двозначні числа `00…99`: ребро йде від «перших двох цифр» до «останніх
   двох цифр» фрагмента. Скласти ланцюжок з фрагментів = пройти графом,
   жодного разу не повторивши ребро (**ейлерів шлях**).

2. **Ідеальний випадок.** Якщо в кожному вузлі кількість вхідних і вихідних
   ребер однакова (за винятком щонайбільше двох вузлів — початку й кінця
   шляху), і граф зв'язний — ейлерів шлях існує і використовує **всі**
   фрагменти. Алгоритм Хієрхольцера будує такий шлях за лінійний час.

3. **Реальні дані.** Якщо граф незбалансований, провести один ланцюжок через усі фрагменти
   неможливо в принципі — незалежно від порядку. У такому разі скрипт:
   - через **min-cost flow** знаходить мінімальну кількість фрагментів,
     які потрібно відкласти, щоб решта графа стала збалансованою;
   - алгоритмом Хієрхольцера будує сам ланцюжок на збалансованому залишку;
   - якщо після відкидання зайвих фрагментів граф розпався на кілька
     непов'язаних шматків, береться найбільший;
   - оскільки оптимальних рішень може бути декілька, крок повторюється
     `TRIALS` разів (за замовчуванням 300, з випадковим порядком перегляду)
     і зберігається найдовший знайдений ланцюжок.

## Вимоги

- Node.js (без зовнішніх залежностей — лише вбудовані модулі `fs`, `path`)

## Використання

```
node index.js [шлях_до_файлу]
```

Якщо шлях не вказано, скрипт читає `source.txt` **у тій самій директорії,
де лежить сам `index.js`** (відносний шлях, не залежить від того, звідки
запущено команду).

```
node index.js               # читає ./source.txt поруч зі скриптом
node index.js data/my.txt   # читає вказаний файл
```

## Формат вхідного файлу

Звичайний текстовий файл, по одному фрагменту в рядку, лише цифри
(порожні рядки ігноруються):

```
942517
605676
498291
...
```

## Формат виводу

```
Loaded fragments: 142        — скільки фрагментів прочитано
Fragments used: 67 of 142    — скільки увійшло в найдовший ланцюжок
Final number length: 270     — довжина підсумкового числа
Fragment chain:               — фрагменти в порядку з'єднання, через " & "
...
Answer:                       — саме підсумкове число
...
```
